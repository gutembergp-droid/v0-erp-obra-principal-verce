1.2 RELACAO ENTRE OS MODULOS
----------------------------

                         +------------------+
                         |      CUSTO       |
                         |   (EAP Custo)    |
                         +--------+---------+
                                  |
                                  | vinculo via codigo EAP
                                  |
         +------------------------+------------------------+
         |                        |                        |
         v                        v                        v
+----------------+       +----------------+       +----------------+
|  PLANEJAMENTO  |<------|      PBS       |------>|   PRODUCAO     |
|  (Cronograma)  |       |  (Estrutura    |       |    (RDO)       |
|                |       |  Operacional)  |       |                |
+----------------+       +----------------+       +----------------+

Fluxo de dados:
1. CUSTO define a EAP de Custo (estrutura financeira)
2. PRODUCAO propoe a PBS (estrutura operacional)
3. PLANEJAMENTO valida a PBS e define cronograma
4. PBS se vincula a EAP via "Mapa Producao -> Custo"
5. Avancos de PRODUCAO propagam para PLANEJAMENTO e CUSTO

-------------------------

3. SEM CALCULO REAL: SPI, CPI, tendencias sao valores fixos

5. INTEGRACAO SIMULADA: Vinculo PBS x EAP existe mas nao propaga dados


================================================================================
6. OBSERVACOES TECNICAS
================================================================================

6.1 SUPOSICOES FEITAS PELO SISTEMA
----------------------------------

1. F/CD (Fator sobre Custo Direto): Assumido 1.15 como padrao

2. BDI: Assumido 25% como padrao

3. TOLERANCIAS: Assumidas fixas por tipo de servico

4. RITMO: Calculado como media simples (sem sazonalidade)


6.2 REGRAS IMPLICITAS
---------------------

1. STATUS DA PBS:
   - Proposta pela Producao = "em_proposta"
   - Validada pelo Planejamento = "validada"
   - Somente PBS validada entra no cronograma

2. RATEIO DE CUSTOS:
   - PBS pode ratear para multiplas EAPs
   - Soma dos rateios deve ser 100%
   - Fator de conversao aplicado antes do rateio

3. CAMINHO CRITICO:
   - Definido manualmente como flag critico: true
   - Destaque visual em vermelho


6.3 COMPORTAMENTOS AUTOMATICOS
------------------------------

1. TOGGLE GRAFICO/TABELA: Todos os cards com dados densos tem alternancia

2. EXPANSAO DE ARVORE: PBS e EAP sao arvores expansiveis

3. CALCULO DE TOTAIS: Somatorios na EAP sao automaticos

4. STATUS POR COR: Verde (OK), Amarelo (Atencao), Vermelho (Critico)
